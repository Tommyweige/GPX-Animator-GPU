# Third-party service terms

Google Places data is provided under Google's Maps Platform terms and requires
the application's required attribution. Users must supply and manage their own
Google Cloud project, key restrictions, billing, quotas, and compliance.

TomTom POI data is provided under TomTom's developer terms. Users must supply
and manage their own TomTom API key and remain within the applicable free
allowance or paid quota. The app prefers Places Search v3 and keeps the legacy
endpoint only as a compatibility fallback.

Foursquare results, including Premium rating/count/popularity fields, require
the user's own plan and must follow Foursquare attribution, cache, and usage
guidelines. A Foursquare rating count is not automatically a text-review count.

Overture and OpenStreetMap snapshots are separate local data packs. Users must
retain the release's source licences and attribution (including ODbL for OSM)
and must not treat a snapshot as a guarantee of current business status.

Gateway Pro is an integration contract only; the user or organisation running
the gateway is responsible for its upstream licences, privacy policy, quota,
authentication, and retention.

Google Places is an explicit optional BYOK profile, not an automatic fallback.
The user is responsible for Google Maps Platform terms, required attribution,
billing, quotas, key restrictions, and any downstream display restrictions.

The production app does not call public Overpass or Nominatim services. If a
user adds a private OSM service outside the supported profiles, that service's
fair-use policy and terms apply separately.
